 Etapa 1: Pré-Projeto (Concluída)

Fase dedicada à idealização, levantamento de requisitos, modelagem preliminar do banco de dados relacional e desenho dos fluxos visuais de experiência do usuário (wireframes).

 Etapa 2: Projeto Final & Implementação (Em breve)

Fase de desenvolvimento prático da aplicação utilizando a arquitetura cliente-servidor, integração dinâmica das APIs locais, persistência de dados real e estilização final da interface.
 Stack Tecnológica Confirmada

O SavePoint foi projetado utilizando tecnologias modernas e eficientes para garantir responsividade e persistência de dados estável:

    Backend: Flask + Jinja2 (Python)

    Banco de Dados: PostgreSQL (Modelagem relacional com suporte nativo a relações complexas)
    
    Frontend: HTML5, Bootstrap 5  e JavaScript Vanilla

    Controle de Versão: Git + GitHub 

  Funcionalidades Planejadas (MVP)

    Backlog de Jogos: Cadastro manual de jogos com notas pessoais e estados personalizados (Quero Jogar, Jogando, Zerado, Platinado, Abandonado).

    Sistema de Categorias: Criação de tags personalizadas pelos usuários para organizar a biblioteca de jogos (Relação Muitos-para-Muitos).

    Rastreador de Conquistas: Acompanhamento percentual do progresso de troféus e metas em cada jogo.

    Diário de Runs: Registro histórico de runs de Roguelikes/Speedruns (tempo, causa da morte, resultado).

    Gerenciador de Builds/Saves: Anotações ricas de distribuição de pontos, itens e habilidades para jogos de RPG.

    Rede de Amizades: Possibilidade de adicionar usuários parceiros para comparar estatísticas e progresso de conquistas de forma direta.
